Laravel Testing

Soporte de Laravel para pruebas

Laravel incluye soporte para realización de pruebas con PHPUnit y phpunit.xml ya está configurado para su utilización.

En el directorio tests encontramos dos directorios:

  • Feature Pueden probar una solicitud HTTP completa a un punto final. Generalmente, la mayoría de sus pruebas deben ser pruebas de características.
  • Unit. Son pruebas que se centran en una parte muy pequeña y aislada de su código. De hecho, la mayoría de las pruebas unitarias probablemente se centren en un solo método. Estas pruebas no inician su aplicación Laravel y, por lo tanto, no pueden acceder a la base de datos de su aplicación u otros servicios marco.

Test preconfigurados en Laravel

En cada uno de los directorios incluidos en la carpeta tests, tenemos un fichero de ejemplo llamado ExampleTest.php. Además, en el estado de desarrollo de la aplicación marcapersonalFP en el que nos encontramos, debemos tener otros tests asociados a librerías instaladas en fases anteriores del desarrollo, como las correspondientes a la autenticación de los usuarios.

Para ejecutar los test ya creados, ejecutaremos el siguiente comando artisan:

 php artisan test

El resultado de esta ejecución será similar al que se muestra a continuación:

 php artisan test
